# OP-008e: `list_available_domains` misses fact-only domains

## Status: pending

## Contract

- **Goal:** Fix `ListAvailableDomains` (MCP tool `list_available_domains`)
  in `src/Deke.Mcp/Tools/SearchTools.cs` to include fact-only domains, by
  also calling `IFactRepository.GetDomainStatsAsync()` and unioning the
  result with the existing `ISourceRepository`-derived domain list, per
  ADR-0009's resolution.
- **Context budget:**
  - ALWAYS: `docs/GLOSSARY.md`, `overhaul/STATE.md`
  - THIS PACKET: `docs/adr/ADR-0009-mcp-list-available-domains-misses-fact-only-domains.md`
    (full, including Resolution — the decision this packet implements);
    `src/Deke.Mcp/Tools/SearchTools.cs` (full — `ListAvailableDomains` is the
    method being changed); `src/Deke.Core/Interfaces/IFactRepository.cs`
    (`GetDomainStatsAsync` signature and the `DomainStats` shape it returns —
    domain name + fact count + last-updated); `src/Deke.Api/Endpoints/FederationEndpoints.cs`
    (the manifest endpoint's existing correct union of `FederationConfig` +
    `IFactRepository.GetDomainStatsAsync` — the pattern to mirror, per
    `docs/PROJECT-MAP.md`'s Federation Endpoints entry, added in OP-004c).
  - Known ties to prior packets: `docs/PROJECT-MAP.md`'s "Search Tools" entry
    (OP-004d) — read it for the exact gap being fixed and the sibling
    `ISourceRepository.GetAllAsync()` call it currently makes alone.
- **Tool budget:** This packet needs a **code-capable agent/session** — Read/
  Edit against `src/Deke.Mcp/Tools/SearchTools.cs`; `dotnet build` to verify
  the change compiles; a way to actually invoke `list_available_domains`
  (e.g. via the MCP server itself, or a direct call into
  `IFactRepository`/`ISourceRepository` against the running Postgres
  instance) to confirm fact-only domains actually appear, not just that the
  code reads correctly.
- **Recommended tier:** Sonnet-class, default thinking — small, well-scoped
  code fix per `docs/TOOLING.md`'s model/thinking tiers table, not a design
  packet (the design decision is already made in ADR-0009).
- **Inputs:** ADR-0009 (accepted, resolution recorded 2026-07-07); OP-004d
  (Deke.Mcp mapping sweep, complete — original finding).
- **Outputs:** `SearchTools.ListAvailableDomains` unions
  `ISourceRepository.GetAllAsync()`-derived domains with
  `IFactRepository.GetDomainStatsAsync()` results, so fact-only domains
  appear in the "Local Domains" output (with fact count, no source count);
  `docs/PROJECT-MAP.md`'s "Search Tools" entry updated to remove the
  ADR-0009 escalation note under Design smells and record the fix instead;
  `overhaul/STATE.md` (new top entry).
- **Done when:** `list_available_domains` returns fact-only domains (with
  fact count, no source count) alongside source-registered domains, verified
  by actually exercising it — e.g. adding a fact under a brand-new domain via
  `add_fact` with no `Source` ever registered for that domain, then
  confirming the new domain shows up in `list_available_domains`'s output;
  `STATE.md` has a new top entry.
- **Human touchpoints:** none expected — this packet implements an already-
  accepted ADR with a fully specified decision.
- **Out of scope:** any other ADR-0009 option (narrowing the documented
  workflow, or leaving as-is — both rejected); OP-004e/OP-004f (Deke.Worker

  ADR-0007's, and ADR-0008's own implementing packets.
