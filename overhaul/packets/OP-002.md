# OP-002: Tooling audit

## Status: COMPLETE (2026-07-07, via Claude Code)

## Contract

- **Goal:** Inventory every MCP server, plugin, and CLI tool available to
  sessions working on DEKE; verify each one actually works against DEKE;
  recommend KEEP / DISCONNECT; write `/docs/TOOLING.md`.
- **Context budget:**
  - ALWAYS: `/overhaul/STATE.md`
  - THIS PACKET: `/overhaul/CHARTER.md`, `OVERHAUL-SKETCH-v0.2.md` §3.5
    (TOOLING.md template) and §4 (tool budget field in the packet contract)
- **Tool budget:** whatever is currently connected/configured in this
  environment — this packet's entire job is to test each one. Likely
  candidates to check: RoslynMcp.Server (C# analysis), Postgres/pgvector
  access, git/GitHub tooling. Explicitly check whether Avient-only tools
  (mssql-rw, SQLcl, windows-mcp) are present in DEKE sessions and flag them
  for disconnection if so — they don't belong here.
- **Recommended tier:** Sonnet-class, default thinking; escalate to extended
  thinking only if a tool's purpose or scope is genuinely ambiguous.
- **Inputs:** OP-001 (workspace must exist).
- **Outputs:** `/docs/TOOLING.md` — inventory table, verification suite,
  model/thinking tier table (see template in the sketch).
- **Done when:** every currently-connected tool has a KEEP/DISCONNECT
  verdict, each with a one-line justification and a pass/fail verification
  check.
- **Human touchpoints:** Mikael approves DISCONNECT verdicts before anything
  is actually removed from config. Phrase as a closed list at the end of the
  session: "Recommend disconnecting: X, Y — confirm?"
- **Out of scope:** modifying naming/glossary; modifying specs; actually
  disconnecting anything without approval.

## Escalation path

If this packet finds a tool that's misconfigured in a way that suggests a
deeper environment/design problem (not just "wrong tool connected"), raise
it as a proposed ADR (type: tooling) rather than fixing it inline.

## Handover

On completion, append a `STATE.md` entry: what was found, what's
KEEP/DISCONNECT, what Mikael needs to confirm, and that OP-003 (glossary
ingestion from `namingissues.md`) is next.
